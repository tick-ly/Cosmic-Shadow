# 宇宙之影：超能力技能系统（物理妥协版）

## 核心设计理念

**超能力 = 物理法则对某一方向的极致妥协**

### 关键概念
1. **代价机制**：每种能力都必须付出相应的物理代价
2. **不稳定性**：能力使用存在失败风险和副作用
3. **限制条件**：能力受物理法则的剩余约束
4. **反噬风险**：过度使用会导致使用者遭受物理法则的反噬

```python
    """技能使用流程"""

    # 检查阶段

    # 评估阶段
    success_rate = calculate_success_rate(unit, skill)
    risk_assessment = assess_risks(unit, skill)
```

```python
def apply_costs(unit, skill):
    """应用技能代价"""

    # 即时代价
    unit.physical_health -= skill.body_strain * 0.1
    unit.mental_energy -= skill.energy_consumption
    unit.stability -= skill.instability_rate * 0.5

    # 累积代价
    unit.reality_debt += skill.reality_debt

    # 副作用判定
    if random() < skill.backlash_risk / 100:
        trigger_backlash(unit, skill)

    # 状态检查
    if unit.reality_debt > unit.debt_threshold:
        trigger_debt_crisis(unit)
```

```python
def assess_risks(unit, skill):
    """风险评估"""

    risk_factors = {
        "身体负担": skill.body_strain,
        "精神消耗": skill.energy_consumption,
        "失败概率": skill.instability_rate,
        "反噬风险": skill.backlash_risk,
        "现实债务": skill.reality_debt
    }

    # 单位当前状态修正
    if unit.physical_health < 30:
        risk_factors["身体负担"] *= 1.5
    if unit.mental_energy < 30:
        risk_factors["精神消耗"] *= 1.5
    if unit.reality_debt > 500:
        risk_factors["现实债务"] *= 2.0

    return risk_factors
```

```python
def evaluate_tactical_decision(unit, skill, situation):
    """战术决策评估"""

    # 收益评估
    tactical_gain = calculate_tactical_value(skill, situation)

    # 风险评估
    risk_assessment = assess_risks(unit, skill)
    total_risk = sum(risk_assessment.values())

    # 决策建议
    if tactical_gain > total_risk * 1.5:
        return "建议使用"
    elif tactical_gain > total_risk:
        return "谨慎使用"
    else:
```

```python
        ("空间折叠", "时间操控"): 1.5,  # 时空一体
        ("概率操控", "因果逆转"): 2.0,  # 量子因果
    }

    base_synergy = 1.0
    if (skill1.name, skill2.name) in synergies:
        base_synergy = synergies[(skill1.name, skill2.name)]

    # 代价叠加
    risk_multiplier = 1.0 + (base_synergy - 1.0) * 0.5

    return {
        "效果倍率": base_synergy,
        "代价倍率": risk_multiplier,
        "综合评估": base_synergy / risk_multiplier
    }
```

```python
class StrategicAI:
    """战略AI系统"""

    def evaluate_combat_situation(self, units, enemies, environment):
        """评估战斗情况"""

        factors = {
            "我方战力": sum(u.power_level for u in units),
            "敌方战力": sum(e.power_level for e in enemies),
            "环境优势": self.evaluate_environment(environment),
            "能力契合度": self.evaluate_skill_synergy(units),
            "风险承受力": self.evaluate_risk_tolerance(units)
        }

        return self.generate_strategy(factors)

    def recommend_skill_usage(self, unit, situation):
        """推荐技能使用"""

        available_skills = unit.active_skills
        recommendations = []

        for skill in available_skills:
            evaluation = self.evaluate_tactical_decision(unit, skill, situation)
            if evaluation in ["建议使用", "谨慎使用"]:
                recommendations.append({
                    "技能": skill.name,
                    "建议": evaluation,
                    "成功率": 100 - skill.instability_rate,
                    "预期收益": skill.tactical_value,
                    "风险等级": self.categorize_risk(skill)
                })

        return sorted(recommendations, key=lambda x: x["预期收益"], reverse=True)
```

## 游戏体验设计

### 玩家体验要点

1. **风险感**：每次使用能力都有明确的失败概率和代价
2. **策略性**：需要权衡风险与收益，不能无限使用强力能力
3. **成长感**：随着熟练度提升，风险降低，效果增强
4. **紧迫感**：现实债务累积会带来时间压力
5. **成就感**：在风险中成功使用强力能力带来巨大满足感

### AI队友行为

```python
class AIUnitBehavior:
    """AI单位行为逻辑"""

    def decide_skill_usage(self, unit, combat_situation):
        """AI技能使用决策"""

        # 性格参数影响决策
        if unit.personality.aggressive:
            risk_tolerance = 0.7
        elif unit.personality.cautious:
            risk_tolerance = 0.3
        else:
            risk_tolerance = 0.5

        # 评估所有可用技能
        for skill in unit.active_skills:
            success_chance = 100 - skill.instability_rate
            expected_value = skill.tactical_value * (success_chance / 100)
            risk_cost = sum(skill.side_effects)

            # 决策逻辑
            if expected_value > risk_cost * risk_tolerance:
                if unit.can_afford_cost(skill):
                    return skill

        return None  # 不使用技能
```

## 总结

### 核心差异对比

| 特征 | 传统RPG系统 | 物理妥协系统 |
|------|------------|--------------|
| 能力本质 | 天赋/魔法 | 物理法则偏离 |
| 使用代价 | 资源消耗 | 存在性代价 |
| 失败可能 | 极低 | 显著 |
| 风险管理 | 简单 | 复杂 |
| 策略深度 | 中等 | 极高 |
| 沉浸感 | 低 | 高 |

### 设计优势

1. **真实性**：基于物理法则的能力体系更加真实可信
2. **平衡性**：高代价自动平衡强力能力
3. **策略性**：风险管理增加游戏深度
4. **沉浸感**：代价和限制增加代入感
5. **可扩展性**：可以基于任意物理法则设计新能力

### 开发优先级

1. **Phase 1**：实现低级血统能力和基础代价系统
2. **Phase 2**：添加中级能力和风险管理系统
3. **Phase 3**：实现高级能力和战略AI
4. **Phase 4**：平衡调整和体验优化
