# 宇宙之影：动态事件系统与局部现实债务方案

## 一、 核心机制：局部现实债务 (Local Reality Debt)

与全局崩溃不同，本作采用**局部数值**设计。
*   **债务浓度 (Debt Concentration)：** 每个节点（战区）拥有独立的“现实债务”数值（0-100）。
*   **浓度积累：** 当单兵/超能力者在该节点强行跨越地形、或在战斗中使用“物理妥协”技能时，该节点的债务浓度上升。
*   **死区化 (Dead Zone)：** 当某节点的债务浓度达到 100 时，该节点发生“现实崩塌”，变为“死区”。死区可能导致：
    *   驻扎在该节点的所有常规部队覆灭。
    *   该节点的所有连线断裂，无法通行。
    *   持续向周围节点蔓延低浓度的债务污染。

## 二、 动态事件触发机制 (Event Triggers)

事件系统与局部债务深度绑定，主要分为三类触发器：

1. **探索触发 (Exploration)：** 单位首次进入“未探索”节点时触发（如：发现旧时代遗迹）。
2. **债务阈值触发 (Debt Threshold)：** 当某节点的债务浓度突破特定阈值（如 30, 60, 90）时触发（如：空间裂隙开启、异次元生物入侵）。
3. **回合/时间触发 (Time/Turn)：** 周期性检测，基于当前节点的属性（地形、危险度）概率触发。

## 三、 事件数据结构模板 (ScriptableObject)

为了方便叙事设计师和策划在 Unity 编辑器中直接填写故事和配置选项，建议使用以下 `ScriptableObject` 数据结构。

团队成员可以将以下代码保存为 `DynamicEventSO.cs`：

```csharp
using System;
using System.Collections.Generic;
using UnityEngine;

namespace ShadowOfTheUniverse.Events
{
    public enum EventTriggerType
    {
        OnExplore,          // 探索时触发
        OnDebtThreshold,    // 债务达到阈值时触发
        RandomTurn          // 回合随机触发
    }

    [CreateAssetMenu(fileName = "NewDynamicEvent", menuName = "Shadow/Dynamic Event")]
    public class DynamicEventSO : ScriptableObject
    {
        [Header("基本信息")]
        public string eventID;
        public string eventTitle;
        [TextArea(3, 10)]
        public string eventDescription;
        public Sprite eventIllustration; // 事件插图

        [Header("触发条件")]
        public EventTriggerType triggerType;
        public int requiredDebtLevel = 0; // 如果是债务触发，需要的最低浓度
        // 可以继续添加地形限制等前置条件...

        [Header("玩家选项")]
        public List<EventChoice> choices = new List<EventChoice>();
    }

    [Serializable]
    public class EventChoice
    {
        public string choiceText;
        [TextArea(2, 5)]
        public string outcomeDescription; // 选择后显示的结果文本

        [Header("结果结算 (Outcomes)")]
        public int debtChange;            // 现实债务的增减（正数为增加债务，负数为净化）
        public int resourceChange;        // 资源的增减
        public bool causeDeadZone;        // 是否直接导致该节点变成死区
        
        // 可以继续添加：获得Buff、触发战斗、解锁新连线等结果...
    }
}
```

## 四、 工作流建议

1. **策划/叙事：** 在 Unity 的 Project 窗口中右键 `Create -> Shadow -> Dynamic Event`，创建新的事件资产。
2. **填表：** 在 Inspector 面板中填写标题、故事描述、配置选项和对应的数值变化（如：选项A增加资源但增加债务，选项B安全但无收益）。
3. **程序：** 编写一个 `EventManager`，在单位移动或回合结束时，遍历这些 `ScriptableObject`，检查触发条件，并在 UI 上弹出对应的事件窗口。
