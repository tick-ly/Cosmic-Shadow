# 宇宙之影：5分钟快节奏战术任务方案

## 一、 核心设计理念：Mission-Based Tactical Sandbox

彻底抛弃“种田”和漫长的扩张期，游戏体验聚焦于**单次高强度任务 (Mission)**。
*   **单局时长：** 不设强制的硬性倒计时，而是通过**流程长度和敌军压力**来控制节奏，确保玩家在熟悉流程的情况下，单局体验在 5 分钟左右完成。
*   **核心循环：** 观察地图拓扑 -> 制定战术路线 -> 执行机动与战斗 -> 使用“物理妥协”破局 -> 撤离或达成目标。
*   **资源机制：** 局内无资源产出。玩家带入局内的“妥协点数”是固定的，用完即止；或者只能通过击杀特定精英敌人/占领特定节点来少量恢复。

## 二、 三大核心任务类型 (Mission Types)

根据玩家操控的单位类型和目标，任务分为三种截然不同的体验：

### 1. 单兵潜入 (Infiltration)
*   **操控单位：** 1-2 名超能力单兵。
*   **任务目标：** 确认敌方重要单位（如指挥官、重型机甲）的位置，并破坏关键单元（如雷达站、护盾发生器、能源核心），为后续的正面部队扫清障碍。**新增目标：在咽喉要道放置“伪装监察器”，为宏观沙盘提供永久视野。**
*   **地图特征：** 节点密集，敌军巡逻路线固定且视野范围明确（引入“警戒范围”概念）。
*   **战术核心：** 避开正面交锋。利用“物理妥协”技能在节点间穿梭。
    *   **新增核心技能【预算直觉 (Predictive Intuition)】：** 消耗妥协点数开启子弹时间，屏幕上出现预测轨迹线，可提前看到敌军未来 3 秒的巡逻路线和狙击手的瞄准红线。代价是极速拉高该区域的现实债务。
*   **失败条件：** 单兵阵亡，或触发警报次数达到上限。

### 2. 正面战争对抗 (Frontline Assault/Defense)
*   **操控单位：** 多支海陆空常规队伍 + 少量单兵。
*   **任务目标：** 坚守阵地 N 个回合/分钟，或在限定时间内攻克敌方重兵把守的堡垒。
*   **地图特征：** 存在明显的“咽喉要道（Choke points）”和开阔地带。
*   **战术核心：** 兵种克制与地形利用。陆军卡住路口，空军进行火力支援。当防线即将崩溃时，单兵使用“物理妥协”（如：大范围轰炸、改变地形阻断敌军）进行清场，但要承担该节点变成“死区”吞噬友军的风险。
*   **失败条件：** 核心阵地失守，或部队全灭。

### 3. 侧线游走与骚扰 (Flanking & Harassment)
*   **操控单位：** 高机动性轻型队伍（如空降兵、特种快艇）。
*   **任务目标：** 截断敌军补给线（摧毁特定连线上的运输队）、占领多个分散的雷达站以削弱敌军主基地防御。
*   **地图特征：** 地图较大，连线复杂，敌军主力移动缓慢但火力强大。
*   **战术核心：** 敌进我退，敌驻我扰。利用高机动性在敌军主力的视野盲区穿插。如果被敌军主力咬住，必须果断使用“物理妥协”切断身后的连线（炸毁桥梁/航道）来逃生。
*   **失败条件：** 任务超时，或游走部队被敌军主力歼灭。

## 三、 关卡数据结构设计 (ScriptableObject)

为了支持这种 5 分钟快节奏任务，我们需要在 Unity 中配置更精确的关卡参数：

```csharp
using System.Collections.Generic;
using UnityEngine;

namespace ShadowOfTheUniverse.Missions
{
    public enum MissionType { Infiltration, Frontline, Harassment }

    [CreateAssetMenu(fileName = "NewMission", menuName = "Shadow/Tactical Mission")]
    public class TacticalMissionSO : ScriptableObject
    {
        [Header("任务简报")]
        public string missionName;
        public MissionType type;
        [TextArea] public string briefingText;
        
        [Header("时间与限制")]
        public float timeLimitSeconds = 300f; // 5分钟倒计时
        public int startingCompromisePoints = 100; // 初始可用的妥协点数
        public int maxAlertLevel = 3; // 潜入任务的最高警戒值

        [Header("地图与目标")]
        public TextAsset mapLayoutJSON; // 预设地图拓扑
        public string targetNodeID; // 目标节点
        
        [Header("敌军 AI 配置")]
        public float enemySpawnRate; // 正面战场的刷兵频率
        public bool enemiesHaveVisionCones; // 潜入任务：敌人是否有视野锥
    }
}
```

## 四、 游戏节奏控制建议

1. **动态压力系统 (Dynamic Pressure)：** 屏幕上方不显示硬性倒计时，而是显示“敌军警戒度”或“现实崩溃进度”。随着玩家在关卡中停留时间变长，敌军会派出越来越强的增援，或者环境灾害会越来越频繁，迫使玩家在 5 分钟左右完成任务并撤离。
2. **绝境引导 (Terminal Guidance) 结局：** 当玩家在 FPS 模式中陷入绝境（血量见底或被重重包围）时，可激活此机制。玩家角色向天空发射高能信标后阵亡，镜头拉回沙盘，一道轨道打击彻底摧毁该节点所有敌军。代价是赢得惨烈胜利的同时，该节点永久变成无法通行的“死区”。
3. **即时制带暂停 (Real-time with Pause)：** 建议采用即时制，但允许玩家按下空格键“战术暂停”，在暂停期间观察局势、下达移动指令或释放超能力。
4. **快速重开 (Quick Restart)：** 高强度的战术任务意味着玩家会经常失败。必须提供“一键秒重开”功能，鼓励玩家不断试错不同的战术路线。
