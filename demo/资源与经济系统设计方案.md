# 宇宙之影：资源与经济系统设计方案

## 一、 核心资源分类 (Resource Types)

为了保持策略深度同时避免过度复杂，建议将资源分为三大类：

1. **常规资源 (Conventional Resources)：**
   *   **建材/工业点 (Industry Points)：** 用于招募常规部队（海陆空队伍）、修建基础设施（如雷达站、防御工事）。
   *   **能源/补给 (Energy/Supplies)：** 维持部队日常消耗，以及支持部队的移动。如果能源耗尽，部队将无法移动或战斗力大减。
2. **战略资源 (Strategic Resources)：**
   *   **异星物质/遗物 (Anomalous Matter)：** 稀有资源，仅在特定节点（如遗迹、高危险度区域）产出。用于招募“单兵/超能力者”，或研发高级科技。
3. **核心机制资源 (Core Mechanic Resource)：**
   *   **妥协点数 (Compromise Points / CP)：** 玩家使用“物理妥协”技能的“蓝条”。
   *   *特殊设定：* 妥协点数不是随时间自然恢复的，而是需要通过“献祭”常规资源、或者在特定节点（如“现实稳定器”）进行转化来获取。这强化了“超能力是有代价的”这一主题。

## 二、 资源的产出与消耗循环 (Economic Loop)

1. **产出 (Generation)：**
   *   **节点产出：** 玩家占领的节点每回合（或每隔一定时间）提供基础资源产出。不同地形产出不同（如：陆地多建材，海洋多能源）。
   *   **建筑增益：** 在节点上修建“矿场”或“发电站”可以提高产出率。
   *   **事件奖励：** 通过动态事件系统获得一次性资源注入。
2. **消耗 (Consumption)：**
   *   **招募与建造：** 一次性消耗建材和能源。
   *   **维护费 (Upkeep)：** 每支部队每回合消耗一定能源。如果入不敷出，部队会陷入“补给匮乏”状态。
   *   **技能释放：** 单兵使用超能力消耗“妥协点数”，同时增加该节点的“现实债务”。

## 三、 经济与“现实债务”的博弈 (Economy vs. Reality Debt)

这是《宇宙之影》经济系统最独特的地方：**经济发展与现实崩溃的拉扯。**

1. **高产出伴随高风险：** 玩家可以修建“超载采集站”，大幅提高资源产出，但代价是该节点每回合都会自动增加“现实债务”。
2. **债务净化成本：** 如果一个节点的债务浓度过高，玩家必须花费大量“能源”和“建材”修建“现实稳定器”来缓慢降低债务。
3. **死区经济学：** 如果一个节点变成“死区”，它将停止所有资源产出，并且驻扎在那里的昂贵部队也会损失。玩家必须在“榨干这个节点”和“保护这个节点”之间做出抉择。

## 四、 数据结构建议 (Data Structure)

为了方便团队实现，建议使用一个全局的 `EconomyManager` 来管理玩家的资源池，并使用字典或专用的类来存储资源数据：

```csharp
// 伪代码示例，供团队参考
public class PlayerResources
{
    public int IndustryPoints;
    public int Energy;
    public int AnomalousMatter;
    public int CompromisePoints;
    
    // 每回合的净产出（产出 - 维护费）
    public int NetIndustry;
    public int NetEnergy;
}
```
