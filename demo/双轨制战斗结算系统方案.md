# 宇宙之影：双轨制战斗结算系统方案

## 一、 核心架构：数据同源，表现分轨

**世界观包装：高维降临 (High-Dimensional Descent)**
玩家在游戏中的本质是一个“高维实体”。在沙盘模式下，玩家在俯视低维宇宙；当玩家点击“亲自指挥”进入 FPS 模式时，实际上是**高维意识降临（附身）**到了某个单兵（容器）身上。如果该单兵在 FPS 模式中阵亡，玩家不会“Game Over”，而是意识被弹回高维（沙盘界面），同时该节点会因为高维容器的破碎而增加大量“现实债务”。

为了让宏观大战略与微观 FPS 射击和谐共存，游戏采用“底层数据互通，表层表现分离”的架构。无论是宏观的“报告式结算”还是微观的“FPS 即时伤害”，它们都基于同一套底层数据（如：单位血量、护甲值、武器威力）。

*   **宏观模式 (Commander Mode)：** 将底层数据转化为**概率和 DPS（每秒伤害）**，进行数学模拟。
*   **微观模式 (FPS Mode)：** 将底层数据转化为**弹道、碰撞体和 Hitbox**，进行物理模拟。

## 二、 模式 A：指挥官报告式结算 (Auto-Resolve / Report System)

当玩家在沙盘界面（不亲自下场）时，节点内发生的战斗采用此系统。

1. **结算逻辑 (Tick-based Simulation)：**
   *   战斗不是瞬间完成的，而是随着沙盘时间（Tick）流逝逐步结算。
   *   系统根据双方的兵种克制（如：反装甲克制重型机甲）、地形加成（如：高地优势）和火力值，计算出每秒的期望伤害。
2. **战报反馈 (Combat Report)：**
   *   UI 上不会显示具体的子弹乱飞，而是显示一个“战况进度条”和“伤亡数字跳动”。
   *   战斗结束后，生成一份战报：歼敌数量、战损比例、消耗的弹药/能源。
3. **指挥官干预：**
   *   玩家虽然不亲自开枪，但可以在沙盘界面使用“物理妥协”技能（如：呼叫轨道打击、改变局部重力）来瞬间改变战局概率。

## 三、 模式 B：FPS 即时伤害系统 (Real-time FPS System)

当玩家选择“亲自带领小队”或“附身”某个单兵进入节点时，游戏无缝切换到第一人称/第三人称射击视角。

1. **结算逻辑 (Physics & Hitbox)：**
   *   完全抛弃概率计算。伤害由真实的物理碰撞决定（子弹是否打中 Hitbox）。
   *   引入 FPS 经典的伤害判定：爆头（弱点伤害倍率）、护甲穿透（跳弹机制）、距离衰减。
2. **“物理妥协”的微观应用：**
   *   在 FPS 模式下，超能力不再是宏观的“改变概率”，而是具体的战术动作。
   *   例如：玩家可以开启“子弹时间（消耗妥协点数）”、使用“空间折跃”闪避火箭弹、或者“扭曲重力”把敌人拉出掩体。
3. **微观反哺宏观：**
   *   玩家在 FPS 模式下的表现（如：无伤暗杀敌方指挥官）会直接反映到沙盘上。如果玩家在 FPS 模式下阵亡，该小队在沙盘上也会被判定为全灭。

## 四、 模式切换的无缝衔接 (Seamless Transition)

为了避免割裂感，我们需要设计一个平滑的过渡机制：

*   **降维打击 (Zoom In)：** 玩家在沙盘上选中一个正在交战的节点，点击“亲自指挥”。镜头从宏观沙盘快速拉近（Zoom In），穿过云层，直接进入 FPS 战场。
*   **升维结算 (Zoom Out)：** FPS 任务完成（或玩家主动撤离）后，镜头拉远回到沙盘。系统将 FPS 模式下的最终存活人数、剩余血量，同步更新到沙盘的 `CombatUnitModel` 中。

## 五、 数据结构建议 (C# 接口设计)

为了让两套系统共用数据，建议提取一个公共的 `ICombatant` 接口：

```csharp
namespace ShadowOfTheUniverse.Combat
{
    // 无论是沙盘上的一个“军团”，还是 FPS 里的一个“士兵”，都实现这个接口
    public interface ICombatant
    {
        float Health { get; set; }
        float Armor { get; }
        float BaseDamage { get; }
        
        // 受到伤害的通用接口
        void TakeDamage(float amount, DamageType type);
    }

    public enum DamageType { Kinetic, Energy, Anomalous }
}
```
