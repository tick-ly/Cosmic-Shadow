# 宇宙之影项目开发总结

## 项目概述

基于《无穷重阻》小说设定的战役策略游戏，核心特色为"物理妥协能力系统"。

## 核心设计理念

### 超能力 = 物理法则对某一方向的极致妥协

- **热力学妥协**：热量转移（违反热力学第二定律）
- **电磁学妥协**：静电聚集（违反库伦定律）
- **力学妥协**：重力干扰（违反万有引力定律）
- **空间妥协**：空间折叠（违反欧几里得几何）
- **时间妥协**：时间操控（违反时间单向性）
- **因果妥协**：因果逆转（违反因果律）

### 核心玩法：风险管理决策

不同于传统RPG的"资源管理"，本游戏的核心是"风险管理"：
- 每次使用能力都有失败概率
- 失败会有反噬和副作用
- 现实债务累积会带来长期后果
- 玩家需要权衡风险与收益

## Plan Agent的关键建议

### 设计优势

1. **独特且自洽的核心机制**
   - 物理妥协系统提供坚实的逻辑基础
   - 现实债务创造天然的长期后果
   - 三级血统提供清晰的成长路径

2. **双层结构的战略深度**
   - 大战略地图 + 战役战斗
   - AI副官系统提供叙事深度
   - 风险管理作为核心循环

### 潜在问题

1. **复杂度过高** ⚠️
   - 当前包含8个相互关联的系统
   - 对于独立开发团队过于庞大
   - 建议：大幅简化

2. **核心循环不够清晰** ⚠️
   - 玩家在每个层级的具体决策不明确
   - 建议：简化核心循环，明确关键决策点

3. **AI系统过度设计** ⚠️
   - 双层AI包含复杂的性格体系
   - 建议：简化为规则型AI

4. **数值系统缺乏锚点** ⚠️
   - 战力值从0到100000+，跨度太大
   - 建议：缩小数值范围

### 优化建议

#### 简化核心循环

```
大战略层（资源管理）
  ↓
战役规划层（队伍配置、风险评估）
  ↓
战术战斗层（风险管理决策）
  ↓
战后结算层（代价支付、现实债务）
  ↓
回到大战略层（资源稀缺性增加）
```

#### 简化系统设计

**优先保留**：
1. 物理妥协能力系统（简化版）
2. 现实债务系统
3. 风险管理战斗
4. 三级血统（简化为低/中/高）

**可以延后或删减**：
- 复杂的AI性格系统 → 2-3种基本类型
- 位卫的特殊成长系统 → 初期不做
- 复杂的协同作战系统 → 初期只支持简单组合
- 大战略地图的多线战役 → 初期只支持单线

#### 数值系统锚定

```python
# 战力值范围
低级血统: 100-500
中级血统: 1000-3000
高级血统: 5000+

# 现实债务
0-300: 安全
300-500: 注意
500-700: 警告
700-900: 危险
900-1000: 极危

# 成功率
低级能力: 85-95%
中级能力: 60-80%
高级能力: 30-50%
```

## 技术实现要点

### 1. 物理妥协系统

```python
class Skill:
    def get_modified_success_rate(self, user):
        rate = self.base_success_rate
        rate -= self.consecutive_uses * 5
        if user.reality_debt > 500:
            rate -= 10
        if user.fatigue > 70:
            rate -= 15
        return max(10, min(95, rate))
```

### 2. 风险分类系统

```python
def classify_risk(success_rate):
    if success_rate >= 85:
        return RiskLevel.SAFE
    elif success_rate >= 70:
        return RiskLevel.LOW
    elif success_rate >= 50:
        return RiskLevel.MEDIUM
    elif success_rate >= 30:
        return RiskLevel.HIGH
    else:
        return RiskLevel.CRITICAL
```

### 3. 现实债务系统

```python
def process_debt_accumulation(unit):
    if unit.reality_debt > 2000:
        return "catastrophic_event"
    elif unit.reality_debt > 1000:
        return "critical_effects"
    elif unit.reality_debt > 500:
        return "warning_effects"
    else:
        return "normal"
```

## 开发难点与解决方案

### 难点1: 风险管理UI设计

**挑战**: 如何清晰展示复杂信息

**解决方案**:
- 使用颜色编码（绿/黄/橙/红）
- 渐进式信息展示
- 鼠标悬停显示详细后果

### 难点2: 数值平衡

**挑战**: 找到"有挑战但公平"的平衡点

**解决方案**:
- 大量测试和调整
- 玩家反馈循环
- 动态难度调整

### 难点3: AI行为设计

**挑战**: 让AI"聪明但不烦人"

**解决方案**:
- 使用简单的决策树
- 性格影响分支选择
- 避免过度优化

## 关键成功因素

### 必须做对的事

1. **第一场战斗必须有趣**
   - 玩家在10分钟内理解并享受风险管理

2. **UI必须清晰**
   - 风险、收益、后果一目了然

3. **平衡必须到位**
   - 高风险高回报，但失败也"玩得起"

4. **节奏必须合理**
   - 债务累积速度适中

5. **成就感设计**
   - 成功使用高风险能力给予强烈反馈

## 可弃用的设计元素

如果开发困难，可以果断放弃：

- ❌ 复杂的AI性格系统 → 简单行为树
- ❌ 大战略地图的开世界探索 → 线性任务
- ❌ 高级血统能力的多样性 → 1-2个代表性能力
- ❌ 多线战役管理 → 单线战役
- ❌ 位卫的特殊成长系统 → 特殊敌人
