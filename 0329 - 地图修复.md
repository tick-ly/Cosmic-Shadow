### 3. MapManager 配置缺失 ⭐ **主要问题**

**代码位置**: `MapManager.cs:163-173`

```csharp
if (terrainTilemap != null)
```

### 4. 缺少必要的资源引用

MapManager 需要以下配置但可能未设置:

#### Tilemap 相关
- ❌ `terrainTilemap` (Tilemap 组件) - 必需
- ❌ `landTile` (TileBase) - 陆地瓦片
- ❌ `seaTile` (TileBase) - 海洋瓦片
- ❌ `coastalTile` (TileBase) - 沿海瓦片
- ⚠️ `spatialFoldingTile` (TileBase) - 空间折叠视觉效果
- ⚠️ `solidSeaAnomalyTile` (TileBase) - 固态海洋异常表现

#### 预制体
- ❌ `capturePointPrefab` (GameObject) - 据点预制体

#### 容器
- ⚠️ `capturePointContainer` (Transform) - 据点容器(可选)

**严重程度**: 🔴 高

## 解决方案

### 步骤 1: 创建 Tilemap 网格系统

1. 在 Hierarchy 中创建 Grid:
   ```
   GameObject → 2D Object → Tilemap → Rectangular
   ```

2. 这会自动创建:
   - Grid (父对象)
   - Tilemap (子对象)

3. 将 Tilemap 重命名为 "TerrainTilemap"

---

### 步骤 2: 创建 Tile 资源

#### 方法 A: 使用 Sprite 创建 Tile
1. 在 Project 窗口创建文件夹:
   ```
   Assets/_Project/Tiles/
   ```

2. 右键 → Create → Tiles → Tile
   - 创建 `LandTile`、`SeaTile`、`CoastalTile` 等

3. 为每个 Tile 分配 Sprite:
   - 在 Inspector 中选择 Tile 资源
   - 拖拽 Sprite 到 "Default Sprite" 字段
   - 设置颜色 (可选)

#### 方法 B: 从 Sprite 自动生成
1. 选择 Sprite 资源
2. Asset → Create → 2D → Tiles → Tile from Sprite

---

### 步骤 3: 创建 CapturePoint 预制体

1. 创建新的 GameObject:
   ```
   Hierarchy → Create Empty
   命名为 "CapturePoint"
   ```

2. 添加组件:
   - `Sprite Renderer` (用于可视化)
   - `StarNode` 脚本
   - `Circle Collider 2D` (可选，用于交互)

3. 配置组件:
   - 设置 Sprite Renderer 的默认 Sprite 和颜色
   - 调整大小 (Scale)

4. 创建预制体:
   - 将 GameObject 从 Hierarchy 拖到 Project 窗口
   - 保存路径: `Assets/_Project/Prefabs/CapturePoint.prefab`

5. 删除场景中的临时 GameObject

---

### 步骤 4: 配置 MapManager 组件

#### 方法 A: 在场景中配置 (推荐)

1. 找到场景中的 MapManager GameObject
   - 如果不存在，创建空对象并添加 MapManager 脚本

2. 在 Inspector 中配置字段:

   **Tilemap 设置**:
   - Terrain Tilemap: 拖入场景中的 Tilemap 对象

   **地形瓦片**:
   - Land Tile: 选择对应的 Tile 资源
   - Sea Tile: 选择对应的 Tile 资源
   - Coastal Tile: 选择对应的 Tile 资源
   - Spatial Folding Tile: (可选)
   - Solid Sea Anomaly Tile: (可选)

   **据点设置**:
   - Capture Point Prefab: 选择 CapturePoint 预制体
   - Capture Point Count: 设置据点数量 (默认 5)
   - Capture Point Container: (可选) 创建空对象作为容器

#### 方法 B: 通过代码自动配置

在 MapManager 脚本中添加自动查找逻辑:

```csharp
private void Awake()
{
    // ... 现有代码 ...

    // 自动查找 Tilemap
    if (terrainTilemap == null)
    {
        terrainTilemap = FindObjectOfType<UnityEngine.Tilemaps.Tilemap>();
        if (terrainTilemap == null)
        {
            Debug.LogError("[MapManager] 场景中未找到 Tilemap 组件！");
        }
    }

    // 自动加载 Tile 资源
    if (landTile == null)
    {
        landTile = Resources.Load<UnityEngine.Tilemaps.TileBase>("Tiles/LandTile");
    }
    // ... 类似处理其他 Tile ...
}
```

## 常见问题

### Q1: 地图生成了但看不到瓦片

**原因**:
- Tilemap 未正确赋值
- Tile 资源未创建或未分配
- 摄像机位置不正确

**解决**:
- 检查 MapManager 的字段赋值
- 确认 Tile 资源有 Sprite
- 调整 Camera 位置和大小

---

### Q2: 据点未生成

**原因**:
- `capturePointPrefab` 为 null
- 没有合适的陆地节点

**解决**:
- 确认预制体已赋值
- 增加 `landRatio` 参数以生成更多陆地
- 检查控制台的警告信息

---

### Q3: 地图生成后立即消失

**原因**:
- Awake/Start 中清空了数据
- 其他脚本干扰

**解决**:
- 检查 `GenerateMap()` 调用时机
- 确保没有重复调用清空方法

---

### Q4: 空间折叠功能不工作

**原因**:
- `spatialFoldingTile` 未设置
- 找不到合适的出口节点

**解决**:
- 设置空间折叠瓦片资源
- 增加 `maxPortalDistance` 参数
- 确保有足够的陆地节点

## 性能优化建议

### 1. Tilemap 优化

- 使用 TilemapRenderer 的 Chunk Mode
- 批量设置瓦片而不是逐个设置:
  ```csharp
  List<TileChangeData> tileChanges = new List<TileChangeData>();
  // ... 收集所有需要设置的瓦片 ...
  terrainTilemap.SetTiles(tileChanges.ToArray());
  ```

### 2. 对象池优化

- 重用 CapturePoint 对象而不是每次销毁重建
- 使用对象池管理可视元素

### 3. 异步生成

对于大地图,考虑异步生成:
```csharp
public IEnumerator GenerateMapAsync()
{
    GenerateNodes();
    yield return null; // 分帧
    GenerateConnections();
    yield return null;
    GenerateVisuals();
}
```

3. 创建 CapturePoint 预制体
