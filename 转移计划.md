## 1. 目标与原则

**目标**：将现有的 Python (Pygame) 验证原型，完整迁移至 Unity 引擎，并确立“大战略沙盘”的底层架构，为后续添加复杂系统（如科研、外交、海量技能）打下坚实基础。

**核心原则（执行者必读）**：
1.  **数据与表现严格分离 (MVC)**：核心数值逻辑（如物理妥协成功率、现实债务计算）必须写成纯 C# 类，**绝对禁止**继承 `MonoBehaviour` 或在 UI 脚本中直接修改核心数据。
2.  **数据驱动设计**：所有静态配置数据（技能属性、星球类型、血统加成）必须使用 `ScriptableObject` 存储，禁止硬编码。
3.  **事件驱动解耦**：模块之间（如战斗结算与 UI 更新）必须通过事件（C# `Action` 或 `UnityEvent`）通信，禁止直接互相引用。

## 5. 后续扩展规范（防架构腐化指南）

当 MVP（最小可行性产品）跑通后，未来添加新系统必须遵守以下流程：

1.  **加新技能**：**严禁改代码**。直接在 Project 窗口右键创建新的 `SkillDataSO` 资产，填入数值即可。
2.  **加新 UI 面板**：新建 Canvas 下的 Prefab，编写独立的 `XXXPanel.cs` 脚本，通过监听全局事件（如 `EventManager.OnTurnEnded`）来更新数据，**严禁**在 UI 脚本里写 `if (debt > 500) { player.hp -= 10; }` 这种业务逻辑。
3.  **加新系统（如科研）**：新建纯 C# 类 `ResearchSystem` 管理数据，新建 `ResearchManager` (MonoBehaviour) 挂在场景中负责初始化和事件注册。
